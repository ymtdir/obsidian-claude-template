`10_WIKI/` を走査して壊れ・抜け・矛盾を洗い出し、ユーザー承認後に修正する。

## 手順

### 1. リンク切れ検出

- `10_WIKI/*.md` を Glob で列挙
- 各ノートから `[[X]]` 形式のリンクを抽出し、`X.md` が `10_WIKI/` に存在しないものを列挙
- 類似ファイル名があれば置換候補として提示（**部分一致または編集距離 2 以内**。候補は 1〜3 件に絞る）

### 2. 孤立ノート検出

- **`10_WIKI/` 内の相互リンクのみで判定**する（`20_DAILY/` 等 vault の他フォルダからの被リンクは対象外）
- どこからもリンクされていないノートを列挙する

### 3. 張り忘れリンクの検出

`zettelkasten-linker` エージェントを **Agent tool の `subagent_type=zettelkasten-linker`** で呼び出し、関連しているがリンクされていないノートペアの候補を取得する。

### 4. 矛盾記述の検出

- **同じ概念を扱うノートの特定**: 手順 1 で取得したファイル名リストに対し、**タイトル共通キーワード + 本文頻出語 Grep + 手順 3 の副産物（関連ペア）** の 3 経路でクラスタ化する
- クラスタ内のノート間で、定義・数値・事実が食い違う箇所を検出する
- 厳密でなくてよい。疑わしいペアを提示してユーザーに判断を委ねる

### 5. 修正

検出結果を次のフォーマットで **1 メッセージに一括提示** する（ページング不要）:

```
## リンク切れ (N 件)
1. `note-a.md:12` `[[削除済みノート]]` → 候補: `[[代替ノート]]`
2. ...

## 孤立ノート (M 件)
3. `orphan-topic.md`
...

## 張り忘れリンク候補 (K 件)
4. `[[A]]` ↔ `[[B]]`: 関連理由
...

## 矛盾記述の候補 (L 件)
5. `X.md` vs `Y.md`: 食い違い内容
...
```

- 承認は **項目番号の指定** で受け取る（例: `1-3, 5, 8-10`）。カテゴリ一括（例: 「リンク切れ全部」）も可
- 承認された項目のみ修正を適用する
- リンク挿入は本文の文脈に自然に埋め込む（末尾一覧にしない）
