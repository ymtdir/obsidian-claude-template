`10_WIKI/` のノート間リンクを点検し、壊れ・抜けを修繕する。古いノートを書いた当時はまだ存在しなかった概念のノートが後から追加されても、参照が貼られないまま放置されがち。本 skill はそれらを定期的に検出して張り直す。

## 主機能

### 1. リンク切れの修正

- `10_WIKI/*.md` を Glob で列挙
- 各ノートから `[[X]]` 形式のリンクを抽出し、`X.md` が `10_WIKI/` に存在しないものを列挙
- 類似ファイル名があれば置換候補として提示（**部分一致または編集距離 2 以内**。候補は 1〜3 件に絞る）

### 2. 張り忘れリンクの追加

`zettelkasten-linker` エージェントを **Agent tool の `subagent_type=zettelkasten-linker`** で呼び出し、関連しているがリンクされていないノートペアの候補を取得する。古いノートから新しく追加された関連ノートへの参照漏れを拾う主力の手順。

## 副次機能

### 3. 孤立ノートの検出

- **`10_WIKI/` 内の相互リンクのみで判定**する（vault の他フォルダからの被リンクは対象外）
- どこからもリンクされていないノートを列挙する

### 4. 矛盾記述の検出

- **同じ概念を扱うノートの特定**: 手順 1 で取得したファイル名リストに対し、**タイトル共通キーワード + 本文頻出語 Grep + 手順 2 の副産物（関連ペア）** の 3 経路でクラスタ化する
- クラスタ内のノート間で、定義・数値・事実が食い違う箇所を検出する
- 厳密でなくてよい。疑わしいペアを提示してユーザーに判断を委ねる

## 修正の適用

検出結果を次のフォーマットで **1 メッセージに一括提示** する（ページング不要）:

```
## リンク切れ (N 件)
1. `note-a.md:12` `[[削除済みノート]]` → 候補: `[[代替ノート]]`
2. ...

## 張り忘れリンク候補 (K 件)
3. `[[A]]` ↔ `[[B]]`: 関連理由
...

## 孤立ノート (M 件)
4. `orphan-topic.md`
...

## 矛盾記述の候補 (L 件)
5. `X.md` vs `Y.md`: 食い違い内容
...
```

- 承認は **項目番号の指定** で受け取る（例: `1-3, 5, 8-10`）。カテゴリ一括（例: 「リンク切れ全部」）も可
- 承認された項目のみ修正を適用する
- リンク挿入は本文の文脈に自然に埋め込む（末尾一覧にしない）
