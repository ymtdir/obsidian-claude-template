トピックを調査・整理し、20_NOTES にツェッテルカステン形式のノートを作成する。

引数: $ARGUMENTS（トピック名、または `00_INBOX/` 内のファイル名）

## 手順

### 1. 入力を判断する

- `00_INBOX/$ARGUMENTS` にファイルが存在すればそれを読み込み、内容を元にする
- ファイルが存在しなければ、$ARGUMENTS をトピックとしてそのまま扱う

### 2. 調査・整理する

- トピックを補足・整理する
  1. まず `20_NOTES/` 内を Grep で関連キーワード検索し、既存の知識を把握する
  2. 不足があれば WebSearch で調査する
  3. Claude の知識で定義や背景を補完する
- 1ノート = 1アイデアの原則に従い、複数トピックが混在している場合は分割を提案する

### 3. 20_NOTES にノートを作成する

- ファイル名はトピックを端的に表す名前にする（例: `Rustの所有権.md`）
- `template.md` の形式に従って作成する
- プロパティ（YAML frontmatter）は使わない
- ファイル名がタイトルになるので `# 見出し` は不要。本文の見出しは `##` から使い、その下の区分けに `###` を使う
- `20_NOTES/` 内を Grep でトピック関連キーワードを検索し、ヒットしたノートがあれば本文中に `[[wikilink]]` を自然に埋め込む
- 関連ノートへのリンクは末尾にまとめず、文脈の中に配置する

### 4. 後処理

- 作成したノートの内容をユーザーに提示する
- 元が `00_INBOX/` のファイルだった場合、削除してよいか確認する
